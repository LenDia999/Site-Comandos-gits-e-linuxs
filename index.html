O que é o Git

Git é um sistema de controle de versão distribuído. Ele registra o histórico de mudanças do seu código e permite trabalhar em paralelo, criar ramificações, voltar no tempo e colaborar com segurança.

$ git init
$ git add .
$ git commit -m "Primeiro commit"
Dica: cada commit descreve o porquê da mudança. Escreva mensagens claras!

O que é o GitHub

GitHub é uma plataforma de hospedagem para repositórios Git na nuvem, com recursos de colaboração, issues, pull requests, wiki, CI/CD e páginas estáticas (GitHub Pages).

Diferenças entre Git e GitHub

GitGitHub
Ferramenta de linha de comando instalada na sua máquina.Serviço online para hospedar repositórios Git.
Gerencia versões localmente e remotamente.Fornece recursos colaborativos e automações.
Ex.: git add, git commit, git merge.Ex.: Pull Requests, Issues, Actions, Pages.

Principais comandos Linux para usar o Git Bash

$ pwd       # caminho atual
$ ls        # listar arquivos
$ ls -la    # listar arquivos ocultos
$ cd pasta  # entrar na pasta
$ cd ..     # voltar um nível
$ mkdir dir # criar pasta
$ touch a.txt # criar arquivo
$ rm a.txt    # remover arquivo
$ rm -r pasta # remover pasta

Principais comandos Linux presentes no README

$ mv nome_atual nome_novo

Como criar um repositório no GitHub

  1. Acesse o GitHub e clique em New.
  2. Defina nome, visibilidade e opcionalmente adicione um README.
  3. Clique em Create repository.

Como ligar (vincular) um projeto existente na máquina

$ git init
$ git add .
$ git commit -m "prepara projeto"
$ git branch -M main
$ git remote add origin https://github.com/usuario/repositorio.git

Como gerar uma versão com Git

$ git tag -a v1.0.0 -m "primeiro release"
$ git push origin v1.0.0

Como enviar da máquina para o GitHub

$ git push -u origin main
